# Learning Journal

웹개발을 공부하는 초보자를 위한 플러그인입니다. 세션 중 겪은 에러/이슈와 새로
배운 개념을, 초보자 눈높이에 맞는 쉬운 말로 요약해서 `learning-log.md` 파일에
날짜별로 계속 쌓아줍니다.

## 구성 요소

| 구성 요소 | 이름 | 역할 |
|---|---|---|
| 스킬 | `learning-journal` | 학습 일지 형식과 저장 규칙을 정의. 사용자가 직접 요청할 때의 진입점 |
| 에이전트 | `learning-journal-writer` | 세션 내용을 검토해 에러/배운 점을 추출하고 초안을 작성 |
| 훅 | `Stop` 이벤트 | 세션이 끝날 때 자동으로 위 흐름을 트리거 |

## 설정

별도의 환경 변수나 외부 서비스 연결이 필요하지 않습니다. 설치하면 바로
작동합니다.

## 사용법

**자동**: 대화 세션이 끝나는 시점에 자동으로 실행됩니다. 그날 세션에 기록할
만한 내용(해결한 에러, 새로 배운 개념)이 있으면 초안을 보여주고, 확인을 받은
뒤에만 `./learning-log.md`(현재 작업 폴더 기준)에 저장합니다. 기록할 내용이
없으면 조용히 넘어갑니다.

**수동**: 정해진 발동 명령어를 말했을 때만 실행됩니다. 아무 대화에서나 느슨하게
반응하지 않도록, 일부러 특정 문구로만 켜지게 만들었습니다.

- "영역전개 학습일지정리" (띄어쓰기나 조사가 조금 달라도 인식하지만, "오늘
  배운거 정리해줘" 같은 일반적인 표현으로는 발동하지 않습니다)

## 저장 방식

- 파일 위치: 프로젝트 루트의 `learning-log.md`
- 날짜별 섹션(`## YYYY-MM-DD`)으로 누적 기록되며, 기존 내용은 지우거나 덮어쓰지
  않습니다.
- **자동 저장이 아니라 확인 후 저장** 방식입니다. 초안을 먼저 보여주고,
  사용자가 승인해야 실제로 파일에 기록됩니다.

## 항목 형식 예시

```markdown
## 2026-07-24

### 오늘 겪은 에러/이슈
- CSS flexbox에서 요소가 정렬 안 됐던 이유: 부모 요소에 display: flex를
  안 줘서 자식 속성이 무시됐음

### 배운 점
- flex-direction과 justify-content의 차이 (행/열 방향과 정렬 축은 다른 개념)

### 참고한 자료
- MDN Flexbox 가이드
```
